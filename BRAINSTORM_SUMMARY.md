### 1. 产品需求文档 (PRD) - 小智

#### 核心用户
- **垂类大模型开发者**: AI应用开发者、算法工程师
- **企业AI应用开发者**: 企业IT部门、SaaS产品团队
- **RAG系统建设者**: 知识库/智能客服系统开发者

#### 用户分层
```
Tier 1: 免费试用用户 (1000次/天)
Tier 2: 开发者版 (10,000次/天)
Tier 3: 企业版 (100,000次/天)
Tier 4: 旗舰版 (定制)
```

#### 鉴权系统
- **API Key认证**: 基础认证方式
- **JWT Token**: 包含用户权限和配额信息
- **请求签名**: HMAC-SHA256，防篡改和重放攻击
- **OAuth 2.0**: 第三方应用集成
- **IP白名单**: 企业级安全

#### 速率限制
| 用户层级 | QPS限制 | 日调用量 | 突发处理 |
|----------|---------|----------|----------|
| 免费试用 | 1 QPS | 1,000 | Token Bucket |
| 开发者版 | 10 QPS | 10,000 | Token Bucket |
| 企业版 | 100 QPS | 100,000 | Token Bucket |
| 旗舰版 | 500 QPS | 定制 | Token Bucket |

#### 反蒸馏保护
- **输出混淆**: 分数扰动算法（可控噪声）
- **水印嵌入**: LSB隐形水印 + 语义水印
- **行为检测**: 
  - 高频批量请求检测
  - 查询模式分析（训练数据提取？）
  - 输出分布异常检测
  - 自动化工具识别
- **分级响应**: 警告→限速→封禁→法律追责

#### 非功能需求
- **性能**: P99延迟 < 500ms，可用性 99.9%~99.99%
- **安全**: SOC 2合规，数据加密，审计日志
- **扩展**: 水平扩展，自动伸缩

### 2. 技术架构方案 - 灵枢

#### 系统架构
```
客户端层 → 接入层(API Gateway) → 安全层 → 服务层 → 模型层 → 数据层
```

#### 鉴权架构
- **多层鉴权**: API Key + JWT + 请求签名
- **权限模型**: RBAC + ABAC 混合
- **配额控制**: 分级配额，实时检查

#### 反蒸馏架构
- **输入层**: 请求指纹识别（IP、行为、内容分析）
- **处理层**: 输出混淆（分数扰动、随机噪声）
- **输出层**: 水印嵌入（隐形水印、语义水印、溯源标识）

#### 技术选型
| 组件 | 选型 |
|------|------|
| API框架 | FastAPI |
| 网关 | Kong/Nginx |
| 缓存 | Redis |
| 数据库 | PostgreSQL |
| 对象存储 | MinIO |
| 监控 | Prometheus + Grafana |
| 部署 | Kubernetes |

#### 实施路线图
- **Phase 1** (2周): 基础功能 + 基础鉴权
- **Phase 2** (2周): 安全加固 + JWT + 输出混淆
- **Phase 3** (2周): 高级保护 + 水印 + 行为检测
- **Phase 4** (2周): 生产就绪 + K8s + 高可用
